# CI/CD Pipeline Documentation

Данный проект использует GitHub Actions для реализации CI/CD пайплайна. Пайплайн настроен для автоматического запуска при пуше или создании Pull Request в ветки `master`, `development` и `uat`.

## Структура пайплайна

Пайплайн состоит из следующих этапов:

1.  **Run Tests**:
    *   Использует окружение на базе PHP 8.3 и SQLite.
    *   Копирует `.env.ci` в `.env`.
    *   Запускает тесты Laravel с генерацией отчета о покрытии (`clover.xml`).
    *   Проверяет покрытие кода с помощью `coverage-checker.php`. Пайплайн завершается ошибкой, если покрытие менее 50%.

2.  **Static Analysis**:
    *   Использует PHPStan (Larastan) на максимальном уровне строгости (`level: max`).
    *   Пайплайн завершается ошибкой при наличии любых ошибок анализа.

3.  **Linting**:
    *   Использует Laravel Pint для проверки стиля кода (PSR-12 / Laravel preset).
    *   В долгоживущих ветках (`master`, `dev`, `uat`) запуск в режиме `--test` (только проверка).
    *   В остальных ветках Pint автоматически исправляет код и делает коммит.

4.  **Deploy Simulation**:
    *   Запускается только после успешного завершения тестов, анализа и линтинга.
    *   В зависимости от ветки выбирается соответствующий файл конфигурации:
        *   `development` -> `.env.dev`
        *   `uat` -> `.env.uat`
        *   `master` -> `.env.prod`
    *   Для ветки `master` настроено использование GitHub Environment `production`, что позволяет настроить обязательный ручной аппрув в интерфейсе GitHub.

5.  **Notify Maintainers**:
    *   Финальный шаг, который уведомляет поддерживающих проект о результате выполнения пайплайна.

## Конфигурационные файлы

*   `.env.dev` - Среда разработки.
*   `.env.uat` - Среда пользовательского тестирования.
*   `.env.prod` - Продуктовая среда.
*   `.env.ci` - Среда для запуска тестов в пайплайне.

## Требования регламента

*   Все переменные используют `camelCase`.
*   Все методы контроллеров и моделей имеют строгую типизацию.
*   Для отступов используются табуляции (согласно настройке в регламенте).
*   Формы сохраняют введенные значения через `old()`.
